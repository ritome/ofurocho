# お風呂帖 — 開発引き継ぎ書

ヒトノワスタジオ第2作 ／ 業務用入浴記録アプリ

---

## 0. プロジェクト概要

**お風呂帖**（おふろちょう）は、居住型ケア施設の入浴記録を業務用に管理する単一HTMLアプリです。

- **スタジオ**: HITONOWA STUDIO（ヒトノワスタジオ）
- **位置づけ**: 第2作（第1作「ワタシノヒトリゴト」と同じ思想を業務文脈に応用）
- **設計思想**: localStorage完結 ／ プライバシー優先 ／ 最小限から育てる
- **第1段階の利用環境**: 施設のiPad 1台（スタッフ専用、Safari起動 file:// 経由）
- **将来構想**: 第3段階でレンタルサーバー導入→複数端末共有（要全面リライト、本書の対象外）

### 設計の北極星

> **スタッフが画面を見て1秒で「これだけタップすればいい」と分かる**こと。

新機能を入れる時は必ずこれを確認する。複雑さは予定編集モード（リーダー用）と設定タブに押し込め、日常の記録画面は徹底的に削ぎ落とす。

## 1. 技術スタック

- **形式**: 単一HTMLファイル（ヒトリゴトと同方針）
- **フレームワーク**: React 18 + Babel スタンドアロン（CDN経由、ビルド不要）
  - 状態管理が複雑（タブ・モーダル・ドロップダウン多数）なのでReact推奨
- **データ保存**: localStorage（JSONシリアライズ）
- **Excel I/O**: SheetJS（CDN）
- **対象ブラウザ**: iPad Safari（主）、Mac/Windowsのモダンブラウザ（副）
- **外部API**: なし（完全オフライン動作）
- **開発環境**: Claude Code v2.1.126 ／ macOS ／ GitHub

CDN例:
```html
<script crossorigin src="https://unpkg.com/react@18/umd/react.production.min.js"></script>
<script crossorigin src="https://unpkg.com/react-dom@18/umd/react-dom.production.min.js"></script>
<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
<script src="https://cdn.sheetjs.com/xlsx-latest/package/dist/xlsx.full.min.js"></script>
```

## 3. データ構造

すべてlocalStorageに保存。キーは `ofurocho.{エンティティ}`。配列型はそのままJSON。

### 3.1 利用者 (residents)

```json
{
  "id": "r-001",
  "名前": "山田 太郎",
  "フリガナ": "ヤマダ タロウ",
  "部屋番号": "102",
  "区分": "通常",
  "入所日": "2025-04-01",
  "在籍中": true,
  "備考": ""
}
```

`区分`: `"通常"` または `"短期"`

### 3.2 スタッフ (staff)

```json
{
  "id": "s-001",
  "名前": "田中 和子",
  "職種": "看護師",
  "サブ役割": ["入浴介助", "処置", "外回り"],
  "在籍中": true
}
```

`職種`: `"看護師"` または `"支援員"`
`サブ役割`の取りうる値:
- 看護師: `"入浴介助"` ／ `"処置"` ／ `"外回り"`
- 支援員: `"入浴介助"` ／ `"更衣"` ／ `"外回り"`

複数保有可（多くのスタッフは2-3個持つ）。

### 3.3 入浴予定 / 記録 (bathPlans)

```json
{
  "id": "p-2026-05-08-AM-r001",
  "日付": "2026-05-08",
  "セッション": "AM",
  "利用者ID": "r-001",
  "状態": "済",
  "記録時刻": "2026-05-08T09:30:00",
  "体制ID": "session-2026-05-08-AM"
}
```

`セッション`: `"AM"` または `"PM"`
`状態`: `"予定"` または `"済"`
`記録時刻`: 「済」になった時のタイムスタンプ。`予定`時はnull。
`体制ID`: 該当セッションの体制ID。体制未設定時はnull。

### 3.4 入浴セッション体制 (sessionTeams)

```json
{
  "id": "session-2026-05-08-AM",
  "日付": "2026-05-08",
  "セッション": "AM",
  "入浴介助": {
    "看護師": "s-001",
    "支援員": ["s-005", "s-006"]
  },
  "処置": {
    "看護師": "s-002"
  },
  "更衣": {
    "支援員": ["s-007"]
  },
  "外回り": {
    "看護師": "s-003",
    "支援員": "s-008"
  },
  "作成時刻": "2026-05-08T08:00:00"
}
```

外回りの看護師は`null`可（支援員のみの日もある）。
更衣の支援員は1〜2名（配列で1個または2個）。

### 3.5 設定 (settings)

```json
{
  "施設名": "ヒトノワ ホーム",
  "入浴目標": { "週": 3, "最低": 2 },
  "警告レベル": { "注意": 3, "警告": 5 },
  "バージョン": "1.0.0",
  "最終バックアップ": "2026-05-07T18:30:00"
}
```

## 4. 画面仕様

3タブ構成: **入浴記録** ／ **記録一覧** ／ **設定**

### 4.1 入浴記録タブ — スタッフ画面（最小設計）

**目的**: 当日の入浴を1タップで記録

**レイアウト**:
- ヘッダー: タイトル「お風呂帖」、本日の日付、「予定を編集 →」ボタン（リーダー用、目立たない）、HITONOWA STUDIO、タブ
- 進捗バー: 「本日 7名済 ／ 残 8名」+ ヒント「タップで「済」に切替（もう一度で取消）」
- カードグリッド: 5列、本日予定の利用者のみ（フィルタ「本日予定のみ」が標準ON、~15名）
- フッター: 本日の体制サマリ（4ポジション、看3/支4等）+「体制を変更 →」ボタン

**カードの状態**:
- **予定**: 背景`--bg-base`、枠2px `--accent-main`、文字 `--text-primary`
- **済**: 背景`--done-bg`、枠2px `--done-border`、文字`--text-soft`、右上に「済」山吹ピル
- **短期**: 左上にリボンバッジ（背景 `--accent-deep`）

**操作**:
- タップ: `予定 ↔ 済` トグル
- メニュー、モーダル、選択モード、バッチボタン、印刷ボタン、振替機能、すべて**なし**
- 「予定を編集」ボタンを押すと予定編集モードへ遷移

**体制が未設定の場合**:
カードタップ時に「本日の体制を先に設定してください」のソフトプロンプト→体制設定画面へ。設定後にタップ復帰。

### 4.2 予定編集モード — リーダー画面

**目的**: 翌日の入浴予定を組む

**入り方**: 入浴記録タブ右上の「予定を編集」ボタン

**レイアウト**:
- モードバナー: タイトル + 「予定編集モード」バッジ、「← 記録に戻る」ボタン、HITONOWA STUDIO
- 日付ナビ + サマリ + 「予定を印刷」ボタン
- セッションタブ: 午前 (N) ／ 午後 (N)（人数バッジ付き、タップで切替）
- 並び替え: 最終入浴が古い順（デフォルト） ／ 部屋番号順 ／ 五十音順
- フィルタ: 未予定のみ ／ 3日以上未浴 ／ 短期 ／ 名前検索
- カードグリッド: 5列、全50名（縦スクロール許容）
- フッター: 「変更は自動保存されます」+ 統計（50名中N名予定）

**カードの状態**:
- **未予定**: クリーム + 細枠 + 最終入浴日表示
- **予定（午前）**: 薄山吹fill + 山吹枠 + 「午前」ピル
- **予定（午後）**: 薄山吹fill + 深山吹枠 + 「午後」ピル
- **今日入浴済み**: くすみクリーム（明日の予定を組む時、今日入った人は優先度低と分かる）

**警告レベル（最終入浴日からの経過日数）**:
- **通常** (0–2日前): 通常表示
- **注意** (3–4日前): 「最終 X日前」が`--accent-deep`色 + 注意三角アイコン
- **警告** (5日以上): 「最終 X日前」が`--warn-red`色 + 赤塗り三角、未予定カードの枠が赤

しきい値は設定で変更可（デフォルト 注意=3、警告=5）。

**操作**:
- タップ（未予定）→ アクティブセッション（午前or午後タブ）に追加
- タップ（予定済）→ 取消（未予定に戻す）
- すべて自動保存
- 「予定を印刷」: 表示中セッションの予定表をA4縦で印刷

### 4.3 記録一覧タブ

**目的**: 入浴履歴の確認・出力

**ビュー切替**: 日 ／ 週 ／ 月

#### 4.3.1 週ビュー（マトリクス）— 既定

利用者×日付の表形式。

**列**: 部屋 ／ 利用者名 ／ 月X/X ／ 火X/X ／ 水X/X ／ 木X/X ／ 金X/X ／ 週計

**セル**:
- 朝バッジ（`--accent-main`塗り、白文字）
- 午バッジ（`--accent-deep`塗り、白文字）
- 空: ダッシュ「—」
- 朝・午両方の場合は縦に並べる

**今日の列**: 薄黄色背景でハイライト

**週計の警告**:
- 0回: 赤色 + 赤塗り三角
- 1回: `--accent-deep`色

**フッター**:
- 日合計の行
- 警告者カード列（週計1回以下の利用者を抽出）

**フィルタ**: 短期のみ ／ 警告のみ（週計1回以下）／ 名前検索

#### 4.3.2 月ビュー（達成率）

**列**: 部屋 ／ 利用者名 ／ 入浴回数 ／ 達成率（バー + %）

**達成率の計算**:
```
達成率 = 月の入浴回数 ÷ 目標 × 100
目標 = 設定の「週N回」× その月の週数
```

**バーの色**:
- 100%以上: 深山吹バー + ✓アイコン
- 50–99%: 山吹バー
- 33–49%: 山吹バー + 注意三角（テキスト深山吹）
- 33%未満: 赤バー + 赤塗り三角（テキスト赤）

**達成状況サマリ（フッター）**:
達成 (100%+) N名 ／ 標準 (50–99%) N名 ／ 注意 (33–49%) N名 ／ 警告 (<33%) N名 ／ 月平均達成率 N%

**フィルタ**: 短期のみ ／ 未達成のみ（警告+注意を抽出）／ 名前検索

#### 4.3.3 半年・年ビュー

月ビューと同じデータを横に並べる。

**列**: 部屋 ／ 利用者名 ／ 11月 ／ 12月 ／ 1月 ／ 2月 ／ 3月 ／ 4月 ／ 半年達成率

各月セルに%表示、色付け（達成/標準/注意/警告のレベル）。期間合計は右端。

#### Excel出力

「Excelで出力」ボタンで表示中の期間（週/月/半年/年）をxlsxファイルに変換。SheetJSで実装。

### 4.4 設定タブ

#### 4.4.1 データ管理

3つのアクションをカード並列で：

- **利用者・スタッフを取り込み（Excel）**: ファイルピッカー → 解析 → 上書き。月初に行う想定。
- **入浴記録をバックアップ（JSON）**: ダウンロード。日次推奨（主アクション、塗りボタン）。
- **バックアップから復元**: JSONアップロード → 上書き（赤枠で警告）。

#### 4.4.2 施設情報

- **施設名**: テキスト入力（印刷物のヘッダーに表示）
- **入浴目標**: 週 N 回（最低 M 回）。達成率計算と警告色の基準。
- **警告レベル**: 注意 X日以上 ／ 警告 Y日以上。デフォルト 3 / 5。

#### 4.4.3 スタッフ管理

- 上部統計: 「登録 60名（看護師 28 ／ 支援員 32）」
- タブ: 看護師 (28) ／ 支援員 (32)
- 検索: 名前検索 + サブ役割フィルタ（入浴介助のみ等）
- リスト: 名前 + サブ役割タグ（保有=塗り、未保有=枠のみ）+ 編集 / 削除
- 「+ スタッフを追加」ボタン

**重要**: 入浴記録の「担当」プルダウンには、サブ役割「入浴介助」を持つスタッフだけが候補に出る。

#### 4.4.4 利用者

- 統計のみ: 「登録 50名（通常 47 ／ 短期 3）」
- 「利用者一覧を表示」ボタンで全名簿モーダル
- **読み取り専用**。更新はExcel取り込みから。

#### 4.4.5 アプリについて + 危険操作

- バージョン、HITONOWA STUDIO、設計思想の一行
- ページ最下部に小さく: 「全データ削除…」（赤枠ボタン、二段階確認）

### 4.5 本日の体制設定（モーダル）

**目的**: 1日のセッション開始時に、4ポジションへスタッフを割り当てる

**入り方**: 入浴記録タブのフッターの「体制を変更」ボタン

**ヘッダー**: 「本日の体制」+ 日付 + AM/PM + 補足「今後の入浴記録に自動で紐づきます」

**セクション**:
1. **入浴介助**: 看護師1 + 支援員2（ドロップダウン3つ）
2. **処置**: 看護師1（ドロップダウン1つ）
3. **更衣**: 支援員1〜2（ドロップダウン1つ + 「+もう1人追加」ボタン）
4. **外回り**: 看護師1（不在チェック可） + 支援員1（ドロップダウン2つ + チェックボックス）

**ドロップダウンのフィルタ**:
- 各ポジションに対応するサブ役割を持つスタッフのみが候補
- 他のポジションで選択済みのスタッフは候補から除外（重複防止）
- 看/支のバッジで職種を視覚化

**フッター**: 「キャンセル」 / 「この体制で保存」+ ヒント「体制が変わったタイミングで再設定」

**AM/PM別管理**:
午前と午後で体制が違うのが普通なので、それぞれ独立に設定。AM/PMの切替は予定編集モードのセッションタブと共通の概念で扱う。

**前回引き継ぎ機能**: 不要（毎日体制が変わるため）

## 5. インタラクション仕様

### 5.1 一日のフロー

1. **前日午後**: リーダーが予定編集モードで翌日のAM/PM予定を組む → 印刷
2. **当日朝**: 主導スタッフがアプリを開く → 「本日の体制（午前）」を設定
3. **入浴中**: 各スタッフが利用者のカードをタップ → 「済」記録
4. **昼休み後**: 「本日の体制（午後）」を設定（午後がある日のみ）
5. **午後の入浴中**: 同様にタップで記録
6. **終業時**: 終了。終業確認ボタンは不要（予定のまま残った人は「未実施」として記録一覧に表示）

### 5.2 状態遷移

```
カード:
  [予定] --tap--> [済]
  [済]   --tap--> [予定]

体制:
  [未設定] --初回設定--> [体制A]
  [体制A]  --「体制を変更」--> [体制B]

予定（編集モード内）:
  [未予定] --tap--> [予定（午前）]   ※午前タブの場合
  [未予定] --tap--> [予定（午後）]   ※午後タブの場合
  [予定]   --tap--> [未予定]
```

### 5.3 自動保存

すべてのユーザー操作は即座にlocalStorageへ書き込む。「保存」ボタンは原則なし（体制設定モーダルだけは明示的に「保存」ボタンあり）。

### 5.4 履歴と削除

- 記録一覧の編集は本書の対象外（読み取りのみ）
- 全データ削除は設定タブの最下部から、二段階確認で実行

## 6. データI/O

### 6.1 Excel取り込み

**シート構成**:
- シート1「利用者」: 部屋番号 / 名前 / フリガナ / 区分 / 入所日 / 備考
- シート2「スタッフ」: 名前 / 職種 / サブ役割（カンマ区切り）

**処理**:
1. ファイル選択
2. SheetJSで解析（XLSX.read）
3. シート別にレコード化
4. 検証: 必須カラム、重複ID（部屋番号 or 名前で識別）
5. 既存の `residents` / `staff` を完全に上書き
6. 取り込み履歴をsettingsに記録

**注意**: 入浴記録は上書きしない（独立して保持）

### 6.2 JSONバックアップ

```js
const backup = {
  version: "1.0.0",
  exportedAt: new Date().toISOString(),
  data: {
    residents: [...],
    staff: [...],
    bathPlans: [...],
    sessionTeams: [...],
    settings: {...}
  }
};
const blob = new Blob([JSON.stringify(backup, null, 2)], { type: 'application/json' });
const filename = `ofurocho-backup-${date}.json`;
// ダウンロードリンクを生成してclick
```

### 6.3 復元

1. JSONファイル選択
2. パース → スキーマ検証（version、必須キー）
3. 確認モーダル（「現在のデータが上書きされます」）
4. 全localStorageキーを上書き
5. ページをリロード

## 7. 印刷仕様

`window.print()` を使用、CSS `@media print` でレイアウト切替。

### 7.1 入浴予定表（予定編集モードから）

**A4縦** ／ 白黒対応

レイアウト:
- ヘッダー: 「入浴予定表」+ 日付 + AM/PM ／ 施設名 + お風呂帖ロゴ
- 担当 / 看護師の手書き欄（空白の下線）
- テーブル: No. / 部屋 / 利用者名 / 区分 / 入浴チェック欄（□）/ 備考
- フッター: 合計人数 + 印刷日時

### 7.2 月次達成率レポート（記録一覧から）

**A4縦** ／ 全50名

レイアウト:
- ヘッダー: 「YYYY年MM月 入浴達成率レポート」+ 施設名
- テーブル: 部屋 / 名前 / 入浴回数 / 達成率（バー + %）
- フッター: 達成/標準/注意/警告の人数 + 月平均達成率

### 7.3 半年・年レポート

**A4横**（月列が増えるため）

レイアウト:
- ヘッダー: 「YYYY年X月〜YYYY年Y月 入浴達成率レポート」
- テーブル: 部屋 / 名前 / 各月%（6個 or 12個）/ 期間達成率
- 50名で複数ページに分かれる場合は氏名繰り返しヘッダー

### 7.4 CSS @media print の方針

```css
@media print {
  /* 画面UI（ヘッダー、タブ、フィルタ、ボタン）を非表示 */
  .no-print { display: none !important; }
  
  /* 印刷用ヘッダーを表示 */
  .print-only { display: block !important; }
  
  /* カラーをモノクロ寄りに */
  body { background: white; color: black; }
  
  /* テーブルの罫線を実線に */
  .matrix-table td, .matrix-table th { border-color: #2d2820; }
  
  /* バーは薄いグレーで（白黒プリンタ対応） */
  .achievement-bar-fill { background: #888 !important; }
  
  /* ページ区切り */
  .page-break { page-break-after: always; }
  
  /* 余白 */
  @page { margin: 15mm; }
  @page :landscape { size: A4 landscape; }
}
```

## 9. 既知の制約

- **localStorage**: 5–10MB上限。年間レコード約780KB、5年以上は余裕
- **単一端末運用前提**。複数端末で同時利用は不可（第3段階の課題）
- **iPad Safari + file://** が主環境。一部API（ファイルダウンロードなど）はプロトコル制限を考慮
- **バックアップ忘れ**は機能で防がない。設定タブで最終バックアップ日を表示するに留める
- **手動Excel取り込み**前提。自動同期はしない

## 12. Claude Code向けメモ

- **スタッフが迷わない設計が第一原則**。新機能を追加する時は「スタッフのタップが増えないか」を必ず確認
- **localStorage完結**を貫く。外部送信なし、サードパーティ依存最小
- **ヒトノワスタジオの世界観**を守る。山吹色の調和、過剰装飾の排除
- **完璧を目指さず、最小限から育てる**。第1段階で動くものを優先
- **予定 ≠ 記録ではない**。今のスキーマでは「予定エントリの状態が予定/済に遷移する」モデル。記録テーブルを別に持たない
- **体制データは記録に紐付ける**だけで、他に副作用なし。集計時に参照される
- **iPad Safariのfile://起動**で動くこと（FileReader, Blob URL等の制約に注意）

実装で迷ったら、最初のスタッフ画面に戻って「ここに何か新しい操作が増えていないか？」を確認すること。
